# Tacview实时遥测设置指南

## 概述

您的飞机仿真系统现在已经完全支持Tacview实时遥测功能！可以在Tacview中实时观看飞机轨迹、姿态和机动过程。

## 功能特性

✅ **实时数据发送**：通过UDP协议向Tacview发送实时飞行数据  
✅ **完整飞行状态**：位置、速度、姿态、过载等所有关键参数  
✅ **多种机动支持**：水平转弯、筋斗、半滚倒转等所有机动类型  
✅ **高精度数据**：0.01秒时间步长，确保轨迹平滑  
✅ **自动CSV保存**：同时保存数据到CSV文件供后续分析  

## 设置步骤

### 1. 安装Tacview
- 下载并安装Tacview（推荐使用专业版以获得完整功能）
- 启动Tacview应用程序

### 2. 启用实时遥测
在Tacview中：
1. 打开 **File** → **Preferences**
2. 选择 **Real-time telemetry** 选项卡
3. 启用 **Enable real-time telemetry**
4. 设置端口为 **42674**（默认端口）
5. 点击 **OK** 保存设置

### 3. 运行仿真程序
```bash
# 编译并运行测试程序
./KinematicManeuverTest.exe
```

程序将自动：
- 启动Tacview遥测连接
- 开始发送实时数据
- 在控制台显示连接状态

## 数据格式

发送到Tacview的数据包括：

| 参数 | 单位 | 描述 |
|------|------|------|
| 时间戳 | 秒 | 仿真时间 |
| 经度 | 度 | 地理经度 |
| 纬度 | 度 | 地理纬度 |
| 高度 | 米 | 海拔高度 |
| 北向速度 | m/s | 北向速度分量 |
| 天向速度 | m/s | 垂直速度分量 |
| 东向速度 | m/s | 东向速度分量 |
| 俯仰角 | 度 | 飞机俯仰角度 |
| 滚转角 | 度 | 飞机滚转角度 |
| 偏航角 | 度 | 飞机偏航角度 |
| 过载 | G | 当前G力 |
| 飞机类型 | - | F-22 |

## 故障排除

### 连接问题
如果Tacview无法接收数据：
1. 检查防火墙设置，确保端口42674未被阻止
2. 确认Tacview实时遥测功能已启用
3. 检查IP地址设置（默认127.0.0.1）

### 数据不显示
如果Tacview连接成功但无数据显示：
1. 确认飞机类型设置为"F-22"
2. 检查坐标范围是否在合理范围内
3. 查看Tacview控制台是否有错误信息

### 性能优化
- 调整更新频率：修改`setUpdateRate()`参数
- 减少数据精度：修改`std::setprecision()`设置
- 优化网络设置：使用本地回环地址127.0.0.1

## 高级功能

### 自定义飞机类型
```cpp
telemetry->sendSimulationData(time, position, velocity, attitude, gForce, "F-35");
```

### 多飞机支持
```cpp
// 创建多个遥测对象
auto telemetry1 = std::make_unique<TacviewTelemetry>("127.0.0.1", 42674, "F-22-01");
auto telemetry2 = std::make_unique<TacviewTelemetry>("127.0.0.1", 42675, "F-22-02");
```

### 网络配置
```cpp
// 自定义目标地址和端口
telemetry->setTarget("192.168.1.100", 42674);
telemetry->setCallsign("CUSTOM-01");
telemetry->setUpdateRate(20.0);  // 20Hz更新频率
```

## 技术细节

### 协议格式
使用简化的Tacview实时遥测协议：
```
Tacview.RealTimeTelemetry.1.0
0,CALLSIGN,AIRCRAFT_TYPE
0,TIMESTAMP,LON,LAT,ALT,NORTH_VEL,UP_VEL,EAST_VEL,PITCH,ROLL,YAW,G_FORCE
```

### 网络配置
- 协议：UDP
- 默认端口：42674
- 数据包大小：< 1024字节
- 更新频率：10Hz（可配置）
